# Q-RoPE Bridge-Task Admissibility v1

## A bridge-task candidate is admissible only if it:
- is closer to positional-encoding behavior than the current transfer families,
- can be stated without reopening hardware,
- can support a bounded symbolic challenger with an explicit frozen basis,
- preserves auditable diagnostics and stop rules,
- does not collapse back into ordinary transfer-family churn.

## Default Rule
- No bridge-task execution opens by default.
- The next move is memo-level bridge-task definition and gate writing only.

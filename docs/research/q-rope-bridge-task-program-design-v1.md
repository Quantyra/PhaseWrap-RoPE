# Q-RoPE Bridge-Task Program Design v1

## Intent
- Shift the next evidence cycle from transfer-count expansion to relevance-bridge collection.

## Program Constraints
- memo-only by default
- local only
- no hardware
- no publication work
- no code reopen until a bridge-task gate is written

## Required Properties
- materially closer to positional-behavior questions than the existing transfer set
- still bounded and auditable
- still compatible with frozen fairness contracts
- should test whether the witness advantage survives under more position-like structure, not just more synthetic relational variety

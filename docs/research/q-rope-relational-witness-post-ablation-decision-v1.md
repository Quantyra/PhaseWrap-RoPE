# Q-RoPE Relational Witness Post-Ablation Decision v1

## Decision
- `BRANCH REMAINS ACTIVE`
- `DO NOT BROADEN`
- `NEXT BOUNDED STEP = SCHEMA-COMPRESSION CONTROL`

## Why the branch remains active
The bounded ablation packet improved branch credibility.

What it ruled out:
- the branch is not a `delta_task` shortcut
- the branch is not narrowly dependent on raw sector means alone
- anti-collapse and forbidden-input conditions still hold after every approved ablation

This matters because the strongest remaining validity concern after split hardening was:
- one narrow approved feature group could still be carrying the whole result

The packet did not support that concern.

## Why the branch still should not broaden
The ablation result exposed a different uncertainty:
- the current approved feature schema is highly redundant

That is good for robustness, but it leaves one unresolved question:
- what is the smallest semantically clean relational subset that still preserves the win?

Until that is understood, broadening into:
- new tasks
- remote execution
- benchmark expansion
would still be premature.

## Next bounded evidence step
### `schema-compression control`
Compare the current full witness schema against two reduced but still approved views:
- `contrasts_only`
- `means_only`

Purpose:
- determine whether the witness win truly lives in relational contrast structure
- or whether the current perfect result survives because the schema is overcomplete

## Why this is the right next move
It is tighter than broad expansion and more informative than another generic robustness rerun.

It directly follows from the ablation packet:
- `minus_group_B` caused the only measurable drop
- all other group removals left the packet perfect

So the next question is not:
- does the branch still work at all?

It is:
- what representation is actually necessary?

## What remains disallowed
- remote execution
- benchmark expansion
- new candidate branches
- larger seed/task matrices

## Bottom line
The witness branch is now the strongest active line in the repository.
It has not yet earned broadening.
The right next move is one more bounded internal-structure test: schema compression.
